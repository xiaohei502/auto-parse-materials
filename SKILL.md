# 自动解析本地资料

## 定位

这个 Skill 提供一套半自动、本地优先的资料解析方案。v1 只处理用户已经手动准备好的本地文件，不自动抓取网页链接，不绕过平台限制，不依赖登录态或反爬处理。

核心流程是：

1. 用户可以直接在对话中粘贴资料，Codex 先识别资料类型，再询问要生成哪种产出。
2. 如果资料较长、需要长期保存、需要保留原文记录或批量处理，用户也可以把原始材料放入 `inputs/`，例如 `.srt`、`.vtt`、`.txt`、`.md`。
3. 对本地文件，Codex 使用 `scripts/parse_material.py` 做预处理，清洗字幕、合并短行、保留必要时间戳。
4. Codex 根据用户选择的产出模板生成结构化 Markdown 笔记。
5. 需要保存时，必须同时保存原文记录、转写稿/字幕稿、预处理稿和最终产出，避免只有总结没有依据。
6. 结果保存到 `outputs/`，或直接在对话中返回。

这套本地方案同时模拟桌面端资料工作台的核心信息架构：

- 知：资源库、总结记录、基础分类、智能分类。
- 行：产出物、提示词、高光笔记、AI 对话、视觉化总结、改写文章、公众号图文。
- 探索：订阅与热门资料的手动收集入口。
- 合集：按项目、课程、主题或作者组织资料。
- 详细页：全文总结、原文细读、高光笔记、逐句字幕、视觉化总结和自定义总结。

日常使用优先阅读中文入口：

- `中文工作台.md`：中文目录对照和推荐对话入口。
- `中文可视化工作台.html`：本地可视化导航页，直接用浏览器打开即可。
- `templates/个人常用-课程学习笔记提示词.md`：用户常用的课程学习笔记整理模板。

## 默认工作流

当用户直接粘贴资料、提供本地资料文件或要求解析资料时：

1. 识别资料类型：字幕、转写稿、普通文本、Markdown、网页复制文本、会议记录、课程材料或混合材料。
2. 如果用户没有指定产出类型，但目标明显是课程、学习、教材、PPT、转写课稿或复习笔记，优先使用 `templates/个人常用-课程学习笔记提示词.md`。
3. 如果用户没有指定产出类型，且无法判断是否为学习笔记，再简短询问希望生成哪种结果：普通总结、详细页、高光笔记、原文细读、公众号图文、课程学习笔记或自定义总结。
4. 如果资料是直接粘贴在对话里的短文本或中等长度文本，直接在对话中处理，不要求用户先保存到 `inputs/`。
5. 如果资料很长、需要复用、需要批量处理，或用户明确要求保存中间结果，再建议放入 `inputs/` 并运行预处理脚本。
6. 对本地 `.srt`、`.vtt`、`.txt`、`.md` 文件优先运行预处理脚本：

   ```bash
   python auto-parse-materials/scripts/parse_material.py <输入文件> --format markdown
   ```

7. 保留原始含义，不编造资料中没有的结论；如果必须推断，明确标注为“推断”。
8. 清理明显噪声：字幕序号、重复时间戳、异常换行、HTML 标签、网页导航文本、重复口癖和无意义空白。
9. 按主题、时间段、说话人、标题或论证流程分段。
10. 如果资料中有明显多个主题或知识点，先提示用户选择是否分步、分开处理，以及分成几部分，不要一次性输出完整笔记。
11. 按用户选择的模板输出笔记。

## 原文与转写稿存档规则

如果用户要求保存资料、建立记录、进入资源库、后续可复查，或资料来自转写内容，Codex 必须保留以下层级：

- 原文记录：保存用户粘贴的原始文本、网页正文、会议记录或未清洗资料。
- 转写稿/字幕稿：保存来自音视频的 `.srt`、`.vtt`、逐字稿或口播稿。
- 预处理稿：保存脚本清洗后的 Markdown 中间结果。
- 最终产出：保存普通总结、详细页、高光笔记、公众号图文等结果。

推荐保存位置：

- `records/originals/`：原文记录，尽量不改写，只做必要元信息包装。
- `records/transcripts/`：转写稿、字幕稿、逐字稿。
- `records/preprocessed/`：预处理后的清洗稿。
- `outputs/`：最终笔记。
- `artifacts/`：二次产出物。

如果资料是用户直接粘贴到对话中的，并且用户要求保存，Codex 应询问或自行生成一个简短文件名，然后把原始粘贴内容保存到 `records/originals/`，再继续处理。

## 直接粘贴资料的交互方式

这是推荐的日常入口。用户可以直接说：

```text
使用 auto-parse-materials 解析下面资料，先问我想生成哪种产出。
```

然后粘贴字幕、转写稿、文章、网页正文、会议记录或课程材料。

如果用户没有指定产出类型，Codex 应先问：

```text
你想生成哪种结果：普通总结、详细页、高光笔记、原文细读、公众号图文、课程学习笔记，还是自定义总结？
```

如果用户已经指定，例如“直接生成详细页 + 高光笔记”，则不需要再询问，直接按对应模板处理。

只有在以下情况才建议用户放入 `inputs/`：

- 资料太长，不适合直接粘贴。
- 用户想保留可复用的预处理文件。
- 用户要批量处理多份材料。
- 用户希望结果进入本地资源库、合集或产出物目录。

## 默认输出

用户使用中文时默认输出中文；英文材料可按材料语言输出，除非用户明确要求翻译。

默认生成 BibiGPT 式 Markdown 笔记：

- 资料信息与解析假设
- 一段式摘要
- 章节大纲
- 时间线
- 关键观点
- 重要术语、人物、工具或日期
- 可继续追问的问题
- 行动项或学习任务
- 清洗后的关键原文摘录

如果用户要求“详细页”或“完整工作台式输出”，优先使用 `templates/detail-page-prompt.md`，生成以下视图：

- 全文总结
- 原文细读
- 口播逐字稿：由字幕优化
- AI 润色：优化排版与错别字
- AI 改写：转换为书面表达
- 高光笔记：默认高光、情绪高点、爆款片段
- 字幕脚本：逐句字幕
- 视觉化总结：截图分析、画面线索、图文草稿
- 自定义总结

资料很长时，先给紧凑摘要，再展开细节。不要整段倒出清洗后的全文，除非用户明确要求。

## 字幕和转写稿处理

对于 SRT、VTT、转写稿或带时间戳文本：

- 只在时间戳有助于定位时保留。
- 把被字幕切碎的短行合并成可阅读句子。
- 按主题或相近时间窗口分组，避免逐句字幕淹没笔记。
- 如果资料中有说话人，提取说话人。
- 如果内容有明显时间顺序，生成时间线。
- 如果用户要学习笔记，把转写稿整理成概念、例子、复习问题和后续练习。

## 手动音视频流程

如果用户只有音频或视频：

1. 先使用外部工具手动获取字幕或转写稿。
2. 把生成的 `.srt`、`.vtt`、`.txt` 或 `.md` 放入 `inputs/`。
3. 再让 Codex 使用本 Skill 清洗并生成 Markdown 笔记。

本 Skill 不负责下载受平台限制的内容，也不处理需要登录、验证码、付费权限或反爬绕过的网页。

## 文件结构

- `inputs/`：原始字幕、转写稿、网页复制文本。
- `records/`：原文记录、转写稿和预处理稿的长期存档。
- `outputs/`：生成后的 Markdown 笔记。
- `templates/`：固定笔记模板和标准提示词。
- `manual/`：手动下载字幕、转写音频、整理材料的操作说明。
- `scripts/`：本地预处理脚本。
- `references/`：可复用输出结构。
- `library/`：本地资源库和总结记录分类。
- `artifacts/`：基于资料生成的产出物。
- `collections/`：合集索引。
- `explore/`：订阅和热门资料的手动收集清单。

## 参考格式

当用户要求 JSON、可复用笔记、知识库导入或固定模板时，读取 `references/output-schema.md`，按其中结构输出。缺失内容保留为空数组或空字符串，不要为了填满字段而编造信息。
